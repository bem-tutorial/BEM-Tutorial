Einleitung

Auf kleineren Broschüren-Websites ist es normalerweise kein großes Problem, wie Sie Ihre Stile organisieren. Du kommst rein, schreibst CSS oder vielleicht sogar SASS. Sie kompilieren alles in einem einzigen Stylesheet mit den Produktionseinstellungen von SASS und fassen es dann zusammen, um alle Stylesheets aus Modulen in einem ordentlichen Paket zusammenzufassen. Bei größeren, komplexeren Projekten ist die Organisation Ihres Codes jedoch auf mindestens drei Arten der Schlüssel zur Effizienz: Sie beeinflusst, wie lange Sie zum Schreiben von Code benötigen und wie viel Code Sie dafür benötigen schreiben Sie und wie viel Ihr Browser laden muss. Dies ist besonders wichtig, wenn Sie mit Themen-Teams zusammenarbeiten und hohe Leistung unerlässlich ist.

Methoden

Es gibt viele Methoden, um den CSS-Fußabdruck zu reduzieren, die Zusammenarbeit zwischen Programmierern zu organisieren und große CSS-Codebasen zu pflegen. Dies ist bei großen Projekten wie Twitter, Facebook und Github offensichtlich, andere Projekte werden jedoch häufig schnell zu einem „riesigen CSS-Dateiformat“.

Vorteile

Blocks, Elements and Modifiers

Modularität

Blockstile sind niemals von anderen Elementen auf einer Seite abhängig, sodass Sie nie Probleme beim Kaskadieren haben. Sie erhalten auch die Möglichkeit, Blöcke von Ihren abgeschlossenen Projekten auf neue zu übertragen.

Wiederverwendbarkeit

Indem Sie unabhängige Blöcke auf unterschiedliche Weise zusammenstellen und auf intelligente Weise wiederverwenden, wird der CSS-Code reduziert, den Sie verwalten müssen. Mit einer Reihe von Stilvorgaben können Sie eine Bibliothek mit Blöcken erstellen, wodurch Ihr CSS sehr effektiv wird.

Struktur

Die BEM-Methode verleiht Ihrem CSS-Code eine solide Struktur, die einfach und leicht verständlich bleibt.
