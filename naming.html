Methoden Block, Element und Modifier Beispiel Vorteile

Benennung

Es ist eine bekannte Tatsache, dass der richtige styleguide die Entwicklungsgeschwindigkeit, das Debugging und die Implementierung neuer Funktionen in altem Code erheblich steigern kann. Leider werden die meisten CSS-Codebasen manchmal ohne Struktur- oder Namenskonventionen entwickelt. Dies führt langfristig zu einer uneinhaltbaren CSS-Codebase. Der BEM-Ansatz stellt sicher, dass jeder, der an der Entwicklung einer Website beteiligt ist, mit einer einzigen Codebasis arbeitet und dieselbe Sprache spricht. Durch die richtige Benennung werden Sie auf die Änderungen im Design der Website vorbereitet.

DOM (Document Object Model)

Der HTML, den sie schreiben wird vom Browser analysiert und vom Browser umgewandelt und im DOM (Document Object Model) angezeigt. Wenn der HTML Code Fehler aufweist wird dieser meistens vom Browser behoben, dieser Code wird dann im DOM vorhanden sein und kann mit CSS formatiert werden. Z.B. wenn in einer Tabelle das Element tbody weggelassen wird, ist das der Fall. DOM wird vom World Wide Web Consortium definiert. Eine Implementierung, die der DOM-Spezifikation genügt, besteht im Sinne der objektorientierten Programmierung aus einem Satz von Klassen zusammen mit deren Methoden und Attributen. Zudem stellt DOM eine Spezifikation der Programmierschnittstelle dar, welche die XML und HTML Elemente in einer Baumstruktur darstellt.

Block

Schließt eine eigenständige Einheit ein, die eigenständig aussagekräftig ist. Während Blöcke verschachtelt sein können und miteinander interagieren, bleiben sie bedeutungsmässig gleich. Es gibt keine Rangfolge oder Hierarchie. Ganzheitliche Entitäten ohne DOM-Darstellung (z. B. Controller oder Modelle) können ebenfalls Blöcke sein.

HTML

Jeder DOM-Knoten kann ein Block sein, wenn er einen Klassennamen akzeptiert.

CSS

  • Verwenden Sie nur den Klassennamen-Selektor
  • Kein Tag-Name oder IDs
  • Keine Abhängigkeit von anderen Blöcken / Elementen auf einer Seite

Benennung

Blocknamen können aus lateinischen Buchstaben, Ziffern und Bindestrichen bestehen. Um eine CSS-Klasse zu bilden, fügen Sie ein kurzes Präfix für den Namensraum hinzu.

Element

Teile eines Blocks und haben keine eigenständige Bedeutung. Jedes Element ist semantisch an seinen Block gebunden

HTML

Jeder DOM-Knoten innerhalb eines Blocks kann ein Element sein. Innerhalb eines gegebenen Blocks sind alle Elemente semantisch gleich.

CSS

  • Verwenden Sie nur den Klassennamen-Selektor
  • Kein Tag-Name oder IDs
  • Keine Abhängigkeit von anderen Blöcken / Elementen auf einer Seite

Benennung

Elementnamen können aus lateinischen Buchstaben, Ziffern, Bindestrichen und Unterstrichen bestehen. Die CSS-Klasse besteht aus Blocknamen und zwei Unterstrichen plus Elementnamen

Modifikator

Flags auf Blöcken oder Elementen. Verwenden Sie sie, um Aussehen, Verhalten oder den Status zu ändern.

HTML

Modifier ist ein zusätzlicher Klassenname, den Sie einem Block- / Element-DOM-Knoten hinzufügen. Fügen Sie Modifiziererklassen nur zu Blöcken / Elementen hinzu, die sie modifizieren, und behalten Sie die ursprüngliche Klasse bei

CSS

  • Verwenden Sie nur den Klassennamen-Selektor
  • Kein Tag-Name oder IDs
  • Keine Abhängigkeit von anderen Blöcken / Elementen auf einer Seite

Benennung

Namen von Modifizierern können aus lateinischen Buchstaben, Ziffern, Bindestrichen und Unterstrichen bestehen. Die CSS-Klasse besteht aus dem Namen eines Blocks oder Elements und zwei Bindestrichen: .block-mod oder .block__elem-mod und .block-color-black mit .block-color-red. Leerzeichen in komplizierten Modifikatoren werden durch Bindestrich ersetzt.
